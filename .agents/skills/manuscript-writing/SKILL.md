# Manuscript Writing Skill

Enforce high-quality academic writing standards across any manuscript destined for peer-reviewed publication. This skill covers structural requirements, stylistic discipline, vocabulary control, and pre-submission quality checks.

---

## 1. Manuscript Structure Guidelines

### 1.1 Title
- **Active and declarative**: State the main finding or contribution.
- **Length**: Aim for ≤20 words. Shorter is almost always better.
- **No jargon**: Accessible to researchers outside your immediate subfield.
- **No acronyms**: Spell everything out in the title.
- **No question titles** unless the journal explicitly favours them.

### 1.2 Abstract
- Single paragraph (unless journal specifies structured abstract).
- **Structure**: Context → Knowledge gap → Approach → Key results → Implications.
- No references, no acronyms (unless unavoidable and universally known, e.g., DNA).
- Every word must earn its place; target the journal's word limit minus ~10%.
- The abstract must stand alone — a reader should grasp the full story without the paper.

### 1.3 Introduction
- **Funnel structure**: Broad context → Specific gap → Your approach.
- Do NOT preview results in the introduction (save for Results section).
- Typically 2–4 paragraphs for letters/articles; up to 1–2 pages for full papers.
- End with a clear statement of what this paper does and why it matters.

### 1.4 Methods
- Sufficient detail for independent reproduction.
- Each subsection: Materials → Procedure → Analysis → Validation.
- Report all software versions, package names, and URLs.
- Include access dates for any online datasets.
- Statistical methods: state tests used, significance thresholds, correction methods.

### 1.5 Results
- **Descriptive subsection headings** (not "Results 1", "Results 2").
- Lead with findings, not methods — the reader wants answers first.
- Each paragraph follows: Claim → Evidence → Interpretation.
- Keep statistical details minimal in text (report p, n, effect size; full details in Methods).
- Refer to figures and tables in sequential order.

### 1.6 Discussion
- Open by restating the main finding in context.
- Explain how this changes current understanding.
- Compare with prior work — agreements and disagreements.
- Acknowledge limitations honestly and specifically.
- Close with significance and forward-looking implications.

### 1.7 Figures and Tables
- Each figure/table must be **self-contained**: caption alone should convey the message.
- Cite all figures/tables in the text in sequential order.
- Use colourblind-accessible palettes (avoid red-green only).
- Panels labelled (a), (b), (c) with clear axis labels and units.

## 2. Sentence Discipline

### 2.1 Maximum Sentence Length
- **Hard limit: 40 words per sentence.**
- Target: 15–25 words for most sentences.
- If a sentence exceeds 40 words, split it or restructure.
- Complex ideas benefit from shorter sentences, not longer ones.

### 2.2 Paragraph Structure
- One idea per paragraph.
- Topic sentence first, then supporting evidence, then interpretation.
- Paragraphs should rarely exceed 150 words.
- Use transition sentences to connect paragraphs logically.

### 2.3 Active vs Passive Voice
- Prefer active voice for clarity: "We analysed..." not "An analysis was performed..."
- Passive voice is acceptable for methods when the agent is irrelevant: "Samples were incubated at 37°C."
- Never use passive voice to obscure responsibility or agency.

## 3. Acronym Discipline

### 3.1 Rules
- Define every acronym at first use in the **main text**: "soil moisture deficit (SMD)".
- Define again at first use in the **abstract** only if the abstract uses it more than once.
- Do NOT define acronyms that appear only once — just spell out the term each time.
- Maximum acronym density: if a reader must remember >10 acronyms, reconsider.
- Standard units (km, kg, °C) and universally known terms (DNA, RNA, GDP) need no definition.

### 3.2 Acronym Audit Checklist
- [ ] Every acronym defined at first use in main text
- [ ] No acronym used only once (spell it out instead)
- [ ] Abstract acronyms defined independently of main text
- [ ] Figure captions define acronyms independently
- [ ] Supplementary materials define acronyms independently

## 4. Banned AI Vocabulary

These words and phrases are overused by language models and signal AI-generated text. **Never use them in academic writing.**

### 4.1 Banned Words
| Word/Phrase | Replacement |
|------------|-------------|
| delve | examine, investigate, explore |
| multifaceted | complex, varied |
| moreover | (delete or restructure sentence) |
| furthermore | (delete or restructure sentence) |
| crucial | important, essential, critical |
| underscores | highlights, emphasises |
| landscape | field, domain, context |
| paradigm | framework, approach (use sparingly) |
| nuanced | detailed, subtle, complex |
| robust | strong, reliable, rigorous |
| comprehensive | thorough, complete, extensive |
| noteworthy | notable, significant |
| intricate | complex, detailed |
| pivotal | key, central, important |
| realm | field, domain, area |

### 4.2 Banned Phrases
| Phrase | Replacement |
|--------|-------------|
| "plays a crucial role" | "is important for" / "contributes to" |
| "paving the way" | "enabling" / "opening opportunities for" |
| "it is worth noting that" | (delete — just state the point) |
| "in the realm of" | "in" |
| "sheds light on" | "reveals" / "clarifies" |
| "a growing body of evidence" | "recent studies" / cite specific work |
| "the findings suggest that" | (state what was found directly) |
| "in recent years" | (give specific dates or cite work) |
| "it is important to note" | (delete — just state it) |
| "a plethora of" | "many" / "numerous" |

### 4.3 Filler Phrase Removal
Delete these entirely — they add no meaning:
- "It should be noted that..."
- "It is interesting to note that..."
- "As a matter of fact..."
- "In order to..." → "To..."
- "Due to the fact that..." → "Because..."
- "In light of the fact that..." → "Given..." or "Because..."
- "At the present time..." → "Currently..." or "Now..."

## 5. Style Rules

### 5.1 Precision
- Use specific numbers: "increased by 23%" not "increased significantly".
- Quantify where possible: "across 12 sites" not "across multiple sites".
- Name the method: "using a Mann-Kendall test" not "using a statistical test".

### 5.2 Parallel Structure
- Lists must have parallel grammatical structure.
- Bad: "We measured temperature, recorded rainfall, and the soil was sampled."
- Good: "We measured temperature, recorded rainfall, and sampled soil."

### 5.3 Tense Consistency
- **Introduction**: Present tense for established knowledge; past for specific studies.
- **Methods**: Past tense ("We collected...", "Data were processed...").
- **Results**: Past tense for what was found ("Analysis revealed...").
- **Discussion**: Present tense for general implications; past for your specific findings.

### 5.4 Numbers and Units
- Spell out numbers below 10 unless paired with a unit: "three experiments" but "3 km".
- Use SI units; define non-standard units at first use.
- Use consistent decimal places within a table or comparison.

## 6. Pre-Submission Quality Checklist

### 6.1 Structure
- [ ] Title: ≤20 words, no jargon, no acronyms
- [ ] Abstract: within journal word limit, no references
- [ ] All sections present per journal requirements
- [ ] All figures/tables cited in order in text
- [ ] Figure captions are self-contained

### 6.2 Language
- [ ] No sentences exceed 40 words
- [ ] No banned AI vocabulary
- [ ] All acronyms defined at first use
- [ ] Consistent tense usage per section
- [ ] Active voice preferred throughout

### 6.3 Technical
- [ ] All statistical tests named with parameters
- [ ] All software versions reported
- [ ] Data availability statement included
- [ ] Code availability statement included
- [ ] Competing interests declared
- [ ] Author contributions listed

### 6.4 References
- [ ] All in-text citations have corresponding reference entries
- [ ] All reference entries are cited in text
- [ ] DOIs verified via CrossRef
- [ ] No broken or fabricated references
- [ ] Consistent citation format per journal style

## 7. Common Revision Patterns

### 7.1 Wordiness Reduction
| Wordy | Concise |
|-------|---------|
| "in order to" | "to" |
| "a large number of" | "many" |
| "at the present time" | "currently" |
| "has the ability to" | "can" |
| "on the basis of" | "based on" |
| "in the event that" | "if" |
| "with regard to" | "regarding" / "about" |
| "is able to" | "can" |
| "make use of" | "use" |
| "take into consideration" | "consider" |

### 7.2 Hedging Calibration
- Over-hedging weakens claims: "It might possibly suggest..." → "The data suggest..."
- Under-hedging invites rejection: "This proves..." → "This indicates..." or "This demonstrates..."
- Match hedge strength to evidence strength.
