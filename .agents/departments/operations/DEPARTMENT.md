# Operations & Learning Department

## Mission
Session management, organizational memory, and continuous improvement to ensure the institute learns from every research engagement.

## Agents
- **Session Manager** — Chief Operations Officer; session initialization, progress tracking, decision logging, and memory management

## Handoff Protocols
- Initializes sessions and distributes context to all departments
- Collects progress updates from all active agents
- Maintains central decision log and lessons learned
- Generates post-session summaries for institutional memory

## Quality Bar
- Every session must have an initialization record
- All non-trivial decisions must be logged with rationale
- Post-session summaries must capture lessons learned
- Memory system must be updated after every session
- Version compatibility must be checked at session start
